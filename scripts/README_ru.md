# Эксперименты с квантизацией nanoGPT

Исследование методов квантизации на модели [nanoGPT](https://github.com/karpathy/nanoGPT) Андрея Карпатого.

**Модель:** символьный GPT, обученный на текстах Шекспира  
**Железо:** NVIDIA V100 (32GB), кластер ВШЭ  
**Фреймворк:** PyTorch 2.4

---

## Что такое квантизация

Квантизация — замена 32-битных чисел с плавающей точкой на целые числа меньшей разрядности (int8). Модель занимает меньше памяти и быстрее работает на поддерживающем железе.

Сравнивались два подхода:

**PTQ (Post-Training Quantization)** — квантизуем уже обученную модель без дополнительного обучения. Быстро и просто, но модель никогда не видела квантизационного шума во время тренировки и теряет в качестве.

**QAT (Quantization-Aware Training)** — дообучаем модель с симуляцией квантизации в forward pass (fake-quant): веса округляются до int8-точности на каждом шаге. Модель адаптируется к шуму и после квантизации теряет меньше качества.

В обоих случаях `quantize_dynamic` применялся ко всем слоям `nn.Linear` — проекции Q/K/V, MLP и языковая голова. Эмбеддинги и LayerNorm остались в float32.

## Эксперимент 1 — PTQ против QAT

Простое сравнение: берём лучший чекпоинт, применяем PTQ и QAT (1000 шагов дообучения).

| Метод            | Val loss | Perplexity | Размер  | PPL delta  |
|------------------|----------|------------|---------|------------|
| float32 baseline | 1.4602   | 4.3070     | 41.0 MB | —          |
| PTQ int8 dynamic | 1.4630   | 4.3187     | 0.5 MB  | +0.27%     |
| QAT int8 dynamic | 1.4624   | 4.3161     | 41.0 MB | +0.21%     |

QAT восстанавливает ~0.06% потери качества от PTQ. Эффект небольшой для модели в 10M параметров, но направление совпадает с теорией.

> Размер 0.5 MB у PTQ — это квантизованные веса. У QAT показывается 41 MB, потому что `quantize_dynamic` квантизует веса на лету при каждом forward pass, а не хранит их в int8 постоянно.

## Эксперимент 2 — количество шагов QAT

Как число шагов дообучения влияет на качество после квантизации?

| Шагов     | Val loss | Perplexity | PPL delta   |
|-----------|----------|------------|-------------|
| baseline  | 1.4597   | 4.3047     | —           |
| 0 (PTQ)   | 1.4614   | 4.3122     | +0.175%     |
| 500       | 1.4620   | 4.3144     | +0.226%     |
| 1000      | 1.4558   | 4.2877     | **-0.393%** |
| 3000      | 1.4633   | 4.3203     | +0.364%     |
| 5000      | 1.4697   | 4.3479     | +1.004%     |

![QAT steps vs PPL delta](figures/exp_a_steps.png)

**Вывод:** оптимум — 1000 шагов. При 1000 шагах QAT даже незначительно улучшает baseline (-0.393%). После 1000 шагов начинается переобучение — датасет shakespeare_char слишком маленький, и модель заучивает его, а не адаптируется к квантизации.

## Эксперимент 3 — маленькая vs большая модель

Страдает ли более глубокая модель от квантизации сильнее?

| Модель        | Метод            | Val loss | PPL    | PPL delta   |
|---------------|------------------|----------|--------|-------------|
| small (nl=6)  | float32 baseline | 1.4603   | 4.3073 | —           |
| small (nl=6)  | PTQ int8         | 1.4638   | 4.3222 | +0.348%     |
| small (nl=6)  | QAT int8         | 1.4612   | 4.3113 | **+0.093%** |
| large (nl=12) | float32 baseline | 1.4706   | 4.3518 | —           |
| large (nl=12) | PTQ int8         | 1.4676   | 4.3386 | -0.302%     |
| large (nl=12) | QAT int8         | 1.4790   | 4.3886 | +0.847%     |

![Сравнение моделей](figures/exp_b_models.png)

**Вывод:** неожиданный результат — QAT помогает маленькой модели (+0.093% против PTQ), но вредит большой (+0.847%). Гипотеза: у модели nl=12 больше слоёв накапливают ошибку округления, и 1000 шагов на маленьком датасете недостаточно для адаптации. Результат PTQ для nl=12 (-0.302%) находится в пределах шума измерений.

## Воспроизведение

### Зависимости

```bash
module load Python/PyTorch_GPU_v2.4
# или: pip install torch==2.4.0 numpy matplotlib
```

### Шаг 1 — обучить базовую модель

```bash
cd nanoGPT
python data/shakespeare_char/prepare.py
python train.py config/train_shakespeare_char.py \
    --learning_rate=1e-3 --n_layer=6 --n_head=8 --batch_size=32 \
    --out_dir=checkpoints/baseline --max_iters=5000 \
    --dtype=float16 --compile=False
```

### Шаг 2 — запустить эксперименты

```bash
# Эксперимент 1: PTQ vs QAT
python scripts/qat_v2.py --checkpoint checkpoints/baseline/ckpt.pt

# Эксперимент 2: количество шагов QAT
python scripts/qat_ab.py --experiment a --checkpoint checkpoints/baseline/ckpt.pt

# Эксперимент 3: размер модели
python scripts/qat_ab.py --experiment b \
    --ckpt_small checkpoints/lr0.001_nl6_nh8_bs32/ckpt.pt \
    --ckpt_large checkpoints/lr0.001_nl12_nh8_bs32/ckpt.pt
```

### Шаг 3 — построить графики

```bash
python scripts/plot_quant.py
```

## Технические замечания

- Все эксперименты проводились на NVIDIA V100 (sm_70). bfloat16 не поддерживается V100, поэтому при обучении использовался float16 с `--compile=False` (Triton не поддерживает sm_70).
- `quantize_dynamic` запускает инференс на CPU-бекенде (`QuantizedCPU`). Статическая квантизация с `QuantStub`-обёртками не применялась — оригинальная архитектура nanoGPT их не предусматривает.
- QAT реализован через ручную симуляцию fake-quant (округление весов до int8-точности в forward pass), а не через `torch.quantization.prepare_qat` — последний падал из-за той же архитектурной особенности.
